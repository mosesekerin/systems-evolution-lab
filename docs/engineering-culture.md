# Engineering Culture

This document defines the mindset expected from every engineer in the team.

Engineering culture determines how we approach problems, how we learn, and how we collaborate.

## 1. Understanding Over Memorization

If you cannot explain a system clearly, you do not understand it.

Copying tutorials without understanding the underlying system is not acceptable.

Every engineer must aim to understand the **why**, not just the **how**.

---

## 2. Evidence Over Claims

Work is not considered complete unless there is evidence.

Evidence includes:

* working code
* logs
* system output
* test results
* documentation

Statements like "it works on my machine" are not sufficient.

---

## 3. Confusion Must Be Visible

Confusion is normal.

Silence about confusion is not acceptable.

Engineers are expected to:

* ask questions early
* document blockers
* request help when necessary

Hidden confusion slows the entire team.

---

## 4. Systems Thinking

We do not think only about individual components.

We think about:

* how systems interact
* where failures can occur
* how performance changes under load

Understanding the whole system is more important than understanding isolated parts.

---

## 5. Learning Is Part of the Work

Research, reading documentation, experimenting, and debugging are part of engineering work.

Learning is not separate from building.

---

## 6. Documentation Is Engineering

If knowledge exists only in someone’s head, it does not exist for the team.

All important discoveries must be documented.

---

## 7. Shared Ownership

The system belongs to the entire team.

No part of the system should be understood by only one person.

Knowledge must be shared.
